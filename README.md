## 1. 项目概述

### 1.1 项目背景

脊柱侧弯是一种常见的脊柱畸形疾病，我国青少年脊柱侧弯发病率约为 1%-3%，需要长期、规范的康复训练。传统的康复训练存在以下痛点：

| 痛点 | 说明 |
|------|------|
| 缺乏专业指导 | 患者居家训练时无法得到康复师实时指导 |
| 依从性难跟踪 | 康复师难以准确掌握患者的训练执行情况 |
| 效果评估困难 | 训练效果缺乏数据化、可视化的评估手段 |
| 复诊数据缺失 | 复诊时缺乏完整的训练历史记录作为参考 |

### 1.2 项目目标

HIRS 系统旨在通过智能化技术，为脊柱侧弯患者提供专业、便捷的居家康复训练服务，同时为康复师提供高效的患者管理工具，实现"院外康复，院内指导"的新模式。

### 1.3 核心价值

| 价值 | 说明 |
|------|------|
| **标准化** | PSSE（Physiotherapeutic Scoliosis Specific Exercises）训练方案的标准化管理 |
| **可视化** | 训练动作的视频/动画演示，患者一看就会 |
| **计时化** | 精准的训练计时和进度控制，确保训练质量 |
| **反馈化** | 实时训练反馈和异常记录，康复师随时掌握情况 |
| **数据化** | 完整的训练数据记录和统计分析，辅助康复效果评估 |

### 1.4 术语表

| 术语 | 英文 | 说明 |
|------|------|------|
| PSSE | Physiotherapeutic Scoliosis Specific Exercises | 脊柱侧弯特异性物理治疗锻炼 |
| Cobb角 | Cobb Angle | 脊柱侧弯角度测量金标准 |
| Risser征 | Risser Sign | 骨骼成熟度评估指标 |
| Lenke分型 | Lenke Classification | 脊柱侧弯三维分型系统 |
| 依从率 | Compliance Rate | 患者实际完成训练与计划的比例 |

### 2.3 数据库设计

#### 2.3.1 核心实体关系

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Therapist │────<│  Patient    │────<│TrainingPlan │
│   (康复师)  │     │   (患者)    │     │ (训练计划)  │
└─────────────┘     └─────────────┘     └──────┬──────┘
      │                    │                    │
      │                    │                    │
      ▼                    ▼                    ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Action     │<────│TrainingRecord│    │TrainingAction│
│  (训练动作)  │     │  (训练记录)   │     │(计划内动作)  │
└─────────────┘     └─────────────┘     └─────────────┘
```

#### 2.3.2 核心表结构

**用户表 (users)**
- id, email, phone, password_hash, name, role, created_at, updated_at

**康复师表 (therapists)**
- id, user_id, institution_id, department, title, created_at

**患者表 (patients)**
- id, therapist_id, name, phone, email, gender, birth_date, diagnosis_info, tags, invited_code, created_at

**训练动作表 (actions)**
- id, name, description, category, duration, sets, target_muscles, tips, video_url, image_url

**训练计划表 (training_programs)**
- id, patient_id, therapist_id, name, description, schedule, start_date, end_date, status

**训练记录表 (training_records)**
- id, patient_id, program_id, start_time, end_time, completed, total_duration, pain_level, notes
